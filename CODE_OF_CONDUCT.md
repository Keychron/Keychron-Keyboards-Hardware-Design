# Code of Conduct

## Our Standard

We want this repository to be useful, respectful, and welcoming for everyone who studies, references, or contributes to these hardware design files.

Examples of behavior that help create a positive environment include:
- Being respectful and constructive in issues and pull requests
- Giving specific, actionable feedback
- Assuming good intent and asking clarifying questions before escalating
- Sharing corrections and documentation improvements clearly

Examples of unacceptable behavior include:
- Harassment, discrimination, or personal attacks
- Dismissive, insulting, or hostile communication
- Deliberate misrepresentation of another contributor's work
- Publishing private or sensitive information without permission

## Scope

This Code of Conduct applies to repository discussions, issues, pull requests, and other project spaces managed for this repository.

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, pull requests, or other contributions that do not align with this Code of Conduct.

## Reporting

If you experience or witness unacceptable behavior, contact the project maintainers through the repository's official support or contact channels.
